# Единый бот курьеров

Telegram-бот для управления курьерами на **Node.js + grammY + PostgreSQL**.

Текущий этап — **Раздел 1: каркас + регистрация** (согласие на обработку ПД,
присвоение системного ID, регистрация нового/действующего курьера, городские
поля Москвы и СПб, отправка заявки менеджеру, просмотр/изменение/сброс данных).

## Установка

```bash
npm install
cp .env.example .env      # заполните BOT_TOKEN и DATABASE_URL
npm run migrate           # создаст таблицы в PostgreSQL
npm start                 # запуск бота (long polling)
```

Для разработки: `npm run dev` (авто-перезапуск при изменениях).

## Переменные окружения (.env)

| Переменная        | Назначение                                              |
|-------------------|---------------------------------------------------------|
| `BOT_TOKEN`       | Токен бота от @BotFather (обязательно)                  |
| `DATABASE_URL`    | Строка подключения к PostgreSQL (обязательно)           |
| `MANAGER_CHAT_ID` | Чат менеджеров для заявок на регистрацию (опционально)  |
| `WEBAPP_URL`      | Ссылка на Telegram Mini-App (опционально)               |
| `KIS_ART_LINK`    | Ссылка для создания КИС АРТ, Москва (опционально)        |

## Структура

```
src/
  index.js            — точка входа, запуск polling
  config.js           — конфиг из .env
  db/
    pool.js           — пул PostgreSQL
    schema.sql        — схема таблиц
    migrate.js        — применение схемы (npm run migrate)
  services/
    courier.js        — доступ к данным курьеров
  services/
    reports.js        — отчёты по слотам, экипировка, претензии, обращения
  bot/
    index.js          — сборка бота, маршрутизация
    engine.js         — универсальный пошаговый движок анкет (текст/контакт/фото/видео)
    fields.js         — поля анкеты регистрации (1.4 / 1.5)
    keyboards.js      — клавиатуры
    menu.js           — главное меню + подменю слотов/светофора + «Мои данные»
    notify.js         — уведомления менеджеру (заявки, отчёты, экипировка, претензии)
    flows/            — определения анкет (регистрация, отчёты, экипировка, претензия, тарифы)
    sections/         — информация по проекту (2.6) и рабочие приложения (2.9)
```
